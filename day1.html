Concepts Clés de Git

  • Repository (Repo) : A repository is a storage space that tracks all versions and changes of a project.
  • Branch : A branch is an independent line of development used to work on features or fixes without affecting the main project.
  • Commit : A commit is a snapshot of changes saved in the repository's history at a specific point in time.
  • Merge : A merge combines changes from one branch into another, usually integrating new features into the main branch.
  • Pull : A pull retrieves the latest changes from the remote repository and integrates them into the local workspace.
  • Push : A push sends local commits to the remote repository, making your updates available to others.
  • Fork : A fork creates a personal copy of someone else's repository, allowing you to experiment without affecting the original project.
  • Pull Request (PR) : A pull request is a request to merge your changes from a branch or fork into another branch, often after a team review.
